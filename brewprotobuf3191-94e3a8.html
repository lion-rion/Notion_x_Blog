RioneのRACOON-Aiでgoogle protobuf3.19.1が必要となりました．3.19.1以外は絶対受け付けない感じなのでどうにか入れる必要があります．

以前以下の記事でhomebrewの過去バージョンの使い方を紹介しました。 christina04.hatenablog.com あれからHomebrewも変わり、以前の方法ではうまくいかなくなったため新しい過去バージョンのインストール方法を紹介します。 以下の課題により以前の手法が使えなくなりました。 Homebrew automatically uninstalls old versions of a formula after that formula is upgraded with brew upgrade, and periodically performs additional cleanup every 30 days.

1. brewは過去のバージョンを残さない方針

brewは過去のバージョンを基本残さない方針です．

Homebrew automatically uninstalls old versions of a formula after that formula is upgraded with brew  upgrade, and periodically performs additional cleanup every 30 days.

FAQ — Homebrew Documentation

そのため3.19.1を入れようとしても入れられないんです！なんと！

2. pip経由

protobufはいろんな言語に対応しているのでもちろんRACOON-AiのPythonにも対応しています．そのためpip経由で入れられるみたいなので

$ pip install protobuf==3.19.1

にてインストールしました．

しかし…

protoc単体ではコマンドが無いこととされ，sudoをつけると現れるのですが，make時にはsudoの有無に関わらずcommand not foundになります…もうワケワカメです．

$ protoc 
command not found 

$ protoc --version
permission denied

$ sudo protoc --version
libprotobuf3.19.1

$ make
command not found

pip経由で入れたらpathが通らない的な問題もあるみたいなので

$ pip show protobuf

で場所を調べてpathを通したのですが，これもうまくいきませんでした．

3. githubからzipで

protobufはOSSなのでgithubのreleaseからzip経由でダウンロードできます．

なので3.19.1のreleaseを見つけてダウンロードしたらいいと考えたのですが，

残念ながらMacOS(arm64)版がありませんでした…

解決策 :

本題です．

Homebrewのtapと呼ばれる仕組みを使って入れることができることが判明しました．

tapというのはパッケージ化されていない非公式のものもパッケージとして登録されているかの用にインストールできるというものです．

ここで本来存在していない3.19.1を登録することでインストールが可能になるみたいです．仕組みについては詳しくは理解していません．

では以下の用にコマンドを実行します

1.tap用リポジトリを作成

githubでリポジトリを用意刷る必要があるのでtap-newで用意します

$ brew tap-new lion-lion/taps

2.brewの古いformulaをtapに展開

$ brew extract protobuf lion-lion/taps --version 3.19.1

3.tapからインストール

$ brew install lion-lion/taps/protobuf@3.19.1

これでインストールできました

以降はbrewを同じ動作になります．brew upgradeをするとこの古いformulaもアップデートされるので注意です．

$ protoc --version
libprotoc 3.19.1
